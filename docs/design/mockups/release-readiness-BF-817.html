2. 필터 적용 — 역할 "QA" 선택 (기획 AC-02)

이 시안의 핵심 원칙: 목록은 2건으로 좁혀졌지만 진행률(50%)·배너·stat 타일은 그대로다 — 요약은 항상 전체 fixture 기준(기획 §4.2).

3. 릴리즈 배너 3-state (기획 §4.3 · AC-09 / AC-10)

판정 우선순위: blocked > 0 이 최우선 — 완료율이 9/10 여도 차단이 1건이라도 있으면 반드시 경고 배너다(기획 EC-08, 긍정 배너 오인 금지).

data-banner-state="blocked" — blocked > 0 (최우선)

4. Status 배지 · 인터랙션 상태 (hover / focus / selected)

배지는 색 + 한글 라벨 + 글리프 3중 표기 — 색 단독으로 구분하지 않는다(기획 §13.3, 색각 이상 대응).
아래 hover/focus 는 정적 시뮬레이션(.is-hover / .is-focus 클래스)이다. dev 는 이 클래스를 구현하지 말고 :hover / :focus-visible 의사클래스를 사용한다.

Status 배지 3종 (pending / blocked / done)

○대기 ▲차단 ✓완료

필터 칩 — 기본 / hover / 선택(aria-pressed=true) / 선택+hover / focus-visible

완료 숨기기 토글 — off / on / focus-visible (role=switch + aria-checked)





필터 초기화 버튼 — 기본 / hover / focus-visible

focus ring 규칙 (기획 AC-13 — 키보드 전용 조작)

outline: 2px solid var(--rr-color-accent); outline-offset: 2px; — 모든 인터랙티브 요소 공통. outline: none 절대 금지. Tab 순서 = DOM 순서(역할 → 상태 → 토글 → 초기화 → 목록), tabindex 임의 지정 금지.

5. 빈 상태 A / B (기획 §12 — 문구 고정)

두 상태는 문구 + 초기화 버튼 유무로 구분된다. dev 는 #board[data-view-state] 값 (empty-filtered / empty-fixture)으로 분기 렌더링한다.
⚠️ 빈 상태 A 에서 요약 패널(진행률/배너/stat)이 그대로 남아 있는 것은 버그가 아니라 설계다 (기획 §4.2).

6. 반응형 — 브레이크포인트 768px (기획 §14)

본 mockup 은 실제 @media (max-width: 767px) 쿼리로 구현되어 있다 — 모바일 뷰를 이미지로 흉내 낸 것이 아니다.
확인 방법: 이 파일을 file:// 로 열고 브라우저 창을 768px 미만으로 좁히면 위 섹션 1·2 의 표가 실제로 카드 스택으로 전환된다.
커밋된 release-readiness-BF-817-mobile.png 는 390px 뷰포트에서 캡처한 것이다.
