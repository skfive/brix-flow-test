7 · 키보드 포커스 흐름 · 접근성 데모(명세 §7.1)
  1. 필터 탭(roving tabindex): ←/→·Home/End 로 이동, Enter/Space 로 필터 적용. 활성 탭 1개만 Tab stop.
  2. 목록(role=listbox): ↑/↓·Home/End 로 행 이동, Enter/Space 로 선택. 선택 행 1개만 Tab stop(roving).
  3. 상세 진입(모바일): 항목 선택 시 상세 헤더 업무명(tabindex="-1")으로 포커스 이동 → "← 목록" 복귀 시 직전 선택 행으로 포커스 반환.
  4. 후속 액션: Tab = textarea → 저장 버튼 순서. Ctrl/Cmd+Enter 저장.
  5. 모든 인터랙티브 요소 :focus-visible 2px accent outline(offset 2px). 위험 요약 패널은 읽기 전용이라 Tab stop 아님(스크린리더로는 aria-live 갱신 통지).

Tab 순서(데스크톱): 필터 탭 → 목록 → 후속 액션 textarea → 저장 버튼. 위험 등급은 색+글리프(●▲◷◆)+한글 라벨 3중 인코딩(색맹 안전, WCAG AA 대비 검증 완료 · 명세 §7.2).
