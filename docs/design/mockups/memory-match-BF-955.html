본 시안은 BF-916 시각 명세의 addendum 으로, BF-954 기획이 도입한 방향키 그리드 이동 · 뒤집힌 카드에서 포커스 유지 · 재시작 초기화 UX 만 시각화합니다. 신규 컬러 토큰 0개 — 기존 --color-focus-ring / :focus-visible 를 그대로 재사용합니다. 카드 보드는 CSS 로 시뮬레이션한 정적 시안이며 실제 게임 로직이 아닙니다. 파란 링은 키보드 포커스(outline:3px + offset 2px)입니다.
1. 방향키 그리드 이동 — 포커스 링이 인접 카드로 이동 (planner AC-K01)

메모리 매치

이동 횟수3회
시간00:18
↑ ← ↓ →

활성 카드(인덱스 5, 2행 2열)에 파란 포커스 링. → 오른쪽 카드(6)로, ↓ 아래 카드(9)로 링만 이동하고 카드 상태(hidden)는 바뀌지 않습니다. matched 카드(2·8) 위로도 포커스 이동 가능(AC-K04).

2. 그리드 경계 클램프 — 가장자리에서 바깥 방향키 시 링 불변 (planner AC-K02)

1행 1열(인덱스 0) — ↑ / ←

위·왼쪽 경계 → 링 제자리 유지(순환 이동 없음)

4행 4열(인덱스 15) — ↓ / →

아래·오른쪽 경계 → 링 제자리 유지

3. 포커스 링 × 카드 상태 조합 — 어떤 상태 위에서도 링이 또렷 (§2.2 대비 검증)

A 포커스 + hidden

어두운 뒷면 + 파란 링. 방향키 진입 지점.

B 포커스 + revealed = 뒤집기 후 유지

핵심: Enter/Space 로 뒤집어도 링이 그 카드에 그대로 남습니다(AC-E01).

C 포커스 + matched

초록 글로우 + 파란 링(색상 대비). matched 위 이동 가능, 활성화만 no-op(AC-K04).

4. 재시작 초기화 UX — 진행 중 → idle 로 리셋 (planner §4, AC-R01·R02·R03)
BEFORE — 진행 중(playing)
이동 횟수8회
시간00:32

이동 8회 · 00:32, 일부 matched/revealed. 포커스는 재시작 버튼(링).

AFTER — 재시작 직후(idle)
이동 횟수0회
시간00:00

16장 전부 hidden(새 셔플) · 0회 · 00:00 · 배너 숨김. 활성 인덱스=0(1번 카드, tabindex="0"). 포커스는 재시작 버튼에 유지 — 보드에 링 없음(focus stealing 방지, AC-R03). 보드에는 다음 Tab 진입 시 링 표시.

5. 키보드 조작 레전드 (planner §2~§4)
←↑→↓
포커스 이동 — 4×4 격자에서 인접 카드로 링 이동. 가장자리는 클램프(순환 없음). 카드 상태는 바뀌지 않음.
EnterSpace
카드 뒤집기 — 포커스한 hidden 카드를 뒤집음(클릭과 동일). 뒤집은 뒤에도 포커스 링 유지. revealed/matched/checking 중에는 no-op.
TabShift+Tab
영역 이동 — 보드(활성 카드 1개) ↔ 재시작 버튼. 보드 내부 카드 간 이동은 방향키 전담(roving tabindex).
↻ 재시작
재시작 — 16장 hidden 재셔플 · 이동/시간 0 리셋 · 배너 숨김 · 활성 인덱스 0 리셋. 포커스는 재시작 버튼에 유지.
