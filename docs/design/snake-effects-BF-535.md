# 지렁이게임 배수 아이템 이팩트 효과 기획 명세

- **Jira**: BF-535
- **Epic**: snake 신규 module
- **작성자**: 박기획 (brix-Flow planner)
- **작성일**: 2026-05-14
- **의존 구현**: BF-533 (배수 아이템 spawn·길이 증가·상단 통계 UI)

---

## 1. 개요

BF-533에서 배수 아이템 4종(1×/2×/4×/8×)이 구현되었으나,  
섭취 시 시각 이팩트 효과는 색상 glow 원 표시에만 그치고 있다.

본 명세는 각 배수 아이템 섭취 시 발생하는 **시각 이팩트 요구사항**을 정의하여  
게임 몰입감을 높이고, 고배수 아이템 섭취를 플레이어가 체감할 수 있도록 한다.

### 1.1 현재 상태 (As-Is)

| 배수 | 색상 | 섭취 이팩트 현황 |
|------|------|----------------|
| 1×   | 황금 `#ffcc00` | 없음 (단순 새 food spawn) |
| 2×   | 하늘 `#00cfff` | 없음 |
| 4×   | 보라 `#cc44ff` | 없음 |
| 8×   | 빨강 `#ff4444` | 없음 |

### 1.2 목표 상태 (To-Be)

배수 등급에 비례하는 시각 이팩트를 추가하여  
`1× < 2× < 4× < 8×` 순으로 강도가 증가하는 연출을 제공한다.

|------|----------|----------|-----------|----------|------------|
| 1×   | `#ffcc00` (황금) | 55% | +10 | +1칸 | Low (저강도) |
| 2×   | `#00cfff` (하늘) | 28% | +20 | +2칸 | Medium (중강도) |
| 4×   | `#cc44ff` (보라) | 13% | +40 | +4칸 | High (고강도) |
| 8×   | `#ff4444` (빨강) |  4% | +80 | +8칸 | Extreme (최고강도) |

## 3. 배수 아이템별 이팩트 컨셉 및 상세 명세

### 3.1 1× 아이템 — 황금빛 파문 (Ripple)

| 항목 | 내용 |
|------|------|
| **이팩트 컨셉** | 잔잔한 황금빛 동심원 파문 |
| **시각 효과** | 섭취 셀 중심에서 반투명 황금 동심원 2개가 외부로 퍼지며 사라짐 |
| **지속 시간** | 400ms |
| **파티클 수** | 없음 (파문 원 2개) |
| **화면 플래시** | 없음 |
| **슬로우 모션** | 없음 |
| **화면 흔들림** | 없음 |
| **파문 최대 반경** | 셀 크기(CELL=20px)의 2배 → 40px |
| **파문 색상** | `rgba(255, 204, 0, 0.6)` → `rgba(255, 204, 0, 0.0)` (fade-out) |
| **게임 진행 차단** | 없음 (이팩트 중에도 게임 루프 계속) |

**애니메이션 흐름:**
```
t=0ms   : 파문 원 1 (r=5px), 파문 원 2 (r=10px) 생성
t=200ms : 파문 원 1 (r=30px, opacity 0.3), 파문 원 2 (r=20px, opacity 0.6)
t=400ms : 파문 원 1 (r=40px, opacity 0), 파문 원 2 (r=40px, opacity 0) → 소멸
```

---

### 3.2 2× 아이템 — 하늘빛 파티클 버스트 (Particle Burst)

| 항목 | 내용 |
|------|------|
| **이팩트 컨셉** | 소형 하늘빛 파티클이 방사형으로 산출 후 소멸 |
| **시각 효과** | 섭취 위치에서 8개 소형 하늘색 파티클이 360° 방사형으로 흩어진 뒤 fade-out |
| **지속 시간** | 500ms |
| **파티클 수** | 8개 |
| **파티클 반경** | 5px |
| **파티클 속도** | 60px/500ms (방사 방향으로 이동) |
| **파티클 색상** | `#00cfff` |
| **화면 플래시** | 없음 |
| **슬로우 모션** | 없음 |
| **화면 흔들림** | 없음 |
| **게임 진행 차단** | 없음 |

**파티클 배치:**
- 8개 파티클을 45° 간격으로 배치 (0°, 45°, 90°, 135°, 180°, 225°, 270°, 315°)
- 각 파티클: 초기 위치 = 섭취 셀 중심, 이동 = 해당 각도의 단위 벡터 × 속도

---

### 3.3 4× 아이템 — 보라빛 플래시 + 파티클 폭발 (Flash + Blast)

| 항목 | 내용 |
|------|------|
| **이팩트 컨셉** | 전체 화면 보라 플래시와 중형 파티클 폭발의 동시 연출 |
| **시각 효과** | ① 전체 캔버스 보라 오버레이 플래시 + ② 섭취 위치 중형 파티클 16개 폭발 |
| **지속 시간** | 700ms |
| **파티클 수** | 16개 |
| **파티클 반경** | 6px |
| **파티클 속도** | 80px/700ms |
| **파티클 색상** | `#cc44ff` |
| **화면 플래시** | 있음 |
| **플래시 색상** | `rgba(204, 68, 255, 0.35)` |
| **플래시 지속** | 300ms (fade-in 100ms → 유지 0ms → fade-out 200ms) |
| **슬로우 모션** | 없음 |
| **화면 흔들림** | 없음 |
| **게임 진행 차단** | 없음 |

**오버레이 플래시 타임라인:**
```
t=0ms   : canvas 전체에 보라 오버레이 렌더 시작, opacity=0
t=100ms : opacity=0.35 (peak)
t=300ms : opacity=0 (fade-out 완료) → 오버레이 제거
```

**파티클 배치:**
- 16개 파티클을 22.5° 간격으로 배치

### 3.4 8× 아이템 — 슬로우 모션 + 빨간 플래시 + 대형 폭발 + 흔들림 (Full Spectacle)

| 항목 | 내용 |
|------|------|
| **이팩트 컨셉** | 슬로우 모션 · 전체 화면 빨간 플래시 · 대형 파티클 폭발 · 화면 흔들림의 전체 연출 |
| **시각 효과** | ① 슬로우 모션 + ② 빨간 화면 플래시 + ③ 24개 대형 파티클 폭발 + ④ 캔버스 흔들림 |
| **지속 시간** | 1000ms |
| **파티클 수** | 24개 |
| **파티클 반경** | 8px |
| **파티클 속도** | 100px/1000ms |
| **파티클 색상** | `#ff4444` |
| **화면 플래시** | 있음 |
| **플래시 색상** | `rgba(255, 68, 68, 0.40)` |
| **플래시 지속** | 300ms |
| **슬로우 모션** | 있음: TICK_MS를 240ms(×2배)로 300ms간 한시적 감속 |
| **화면 흔들림** | 있음: canvas translateX/Y ±3px, 200ms, 2회 반복 |
| **게임 진행 차단** | 없음 (슬로우 모션 중에도 게임 루프 유지, 틱 간격만 확장) |

**슬로우 모션 명세:**
```
t=0ms    : TICK_MS ← 240ms (기본 120ms의 2배)
t=300ms  : TICK_MS ← 120ms (원복)
```
- 슬로우 모션 중 플레이어 조작 입력은 정상 수용
- CPU AI 방향 결정도 정상 실행 (다만 틱 간격이 늘어나 실질 속도 감소)

**화면 흔들림 타임라인:**
```
t=0ms   : canvas translateX(+3px)
t=50ms  : canvas translateX(-3px)
t=100ms : canvas translateX(+3px)
t=150ms : canvas translateX(-3px)
t=200ms : canvas translate(0, 0) 원복
```

**오버레이 플래시 타임라인:**
```
t=0ms   : opacity=0
t=100ms : opacity=0.40 (peak)
t=300ms : opacity=0 (fade-out 완료)
```

## 4. 이팩트 트리거 조건

| 트리거 | 조건 | 비고 |
|--------|------|------|
| **player 섭취** | `playerAteFood === true` (tickFull 기준) | 이팩트 발동 |
| **CPU 섭취** | `cpuAteFood === true` | 이팩트 없음 (CPU 섭취는 시각 이팩트 제외) |
| **싱글 플레이 섭취** | `tick()` 에서 `ateFood === true` | player 섭취와 동일 이팩트 |
| **이팩트 실행 중 추가 섭취** | 이팩트 중 새 food 섭취 시 | 이전 이팩트와 병렬 실행 (중첩 허용) |
| **게임 종료 후** | `state.status === "gameover"` | 이팩트 중 게임 종료 시 이팩트 즉시 중단 |
| **일시정지 상태** | `state.status === "paused"` | 이팩트 애니메이션 일시정지 (resume 시 재개) |

## 7. Acceptance Criteria (Given / When / Then)

### AC-1. 배수 아이템 분류 및 이팩트 컨셉 정의

**Given** 지렁이게임 snake 모듈에 BF-533 기준 4종의 배수 아이템(1×/2×/4×/8×)이 존재하고  
**When** 기획 명세 markdown이 작성되면  
**Then** 배수 아이템 4종 각각에 대해 이팩트 컨셉·파티클 수·화면 플래시 여부·슬로우 모션 여부·지속 시간·강도 항목이 표로 정리되어 있다.

**검증 방법**: 명세 문서의 §3 테이블에서 4개 배수 행 × 6개 항목 셀이 모두 채워져 있는지 확인.

---

### AC-2. 트리거 조건 명시

**Given** 배수 아이템 섭취 트리거가 `tickFull()`의 `playerAteFood` 플래그로 판단되고  
**When** 기획 명세를 검토하면  
**Then** player 섭취 / CPU 섭취 / 일시정지 상태 / 게임 종료 후 각 경우에 대한 트리거 조건이 표로 정의되어 있다.

**검증 방법**: 명세 §4 트리거 조건 표에서 5개 이상의 조건 행이 존재하고, 각 행에 조건과 비고가 작성되어 있는지 확인.

---

### AC-3. 디자이너 인계 항목 명시

**Given** 시각 디자인이 필요한 이팩트 요소들이 존재하고  
**When** 디자이너에게 인계할 때  
**Then** 시각 디자인이 필요한 항목에 필수/선택 여부와 디자이너 자유도가 표시되어 있다.

**검증 방법**: 명세 §6 디자이너 인계 항목 표에서 "필수/선택" 열과 "디자이너 자유도" 열이 각 행에 작성되어 있는지 확인.

---

### AC-4. KPI 측정 포인트 정의

**Given** 이팩트 기능이 사용자 경험에 영향을 줄 수 있고  
**When** 명세를 검토하면  
**Then** 각 수용 기준 항목(시나리오 동작·사용자 영향 0·롤백·KPI)에 대한 검증 방법이 1문장씩 정의되어 있다.

**검증 방법**: 명세 §8의 KPI 측정 포인트 표에서 4개 이상의 KPI 항목과 각 측정 방법이 작성되어 있는지 확인.

### 8.2 사용자 영향 0 검증

- **이팩트 중 충돌 판정**: 이팩트 실행은 순수 렌더링 레이어이며 게임 로직(`tickFull`)에 영향을 주지 않아야 함
- **검증**: `tickFull()` 유닛 테스트에 이팩트 관련 코드가 포함되지 않는 것을 확인 (`logic.js` 변경 없음)

## 9. 롤백 시나리오

| 롤백 조건 | 판단 기준 | 롤백 방법 |
|----------|----------|----------|
| **성능 저하** | RAF 드롭이 3프레임 이상 연속 발생 (50ms 초과 프레임 3회↑) | `EFFECT_ENABLED` 상수를 `false` 로 변경 → 이팩트 렌더링 전체 비활성화 |
| **슬로우 모션 버그** | 8× 섭취 후 TICK_MS가 120ms로 복귀하지 않는 현상 발생 | `slowMoActive = false; TICK_MS = 120;` 강제 리셋 또는 슬로우 모션 기능만 제거 |
| **화면 흔들림 레이아웃 깨짐** | 흔들림 중 HUD/오버레이 위치 이탈 발생 | 캔버스 흔들림 대신 CSS `filter: brightness(1.5)` 대체 연출로 교체 |
| **이팩트 중첩 메모리 누수** | 파티클 배열 크기가 200개 이상 축적되는 현상 | `activeEffects` 최대 크기 제한 (20개) + 강제 flush 로직 추가 |
| **전체 이팩트 롤백** | QA에서 치명적 버그 발견 시 | `game.js`의 `triggerEffect()` 호출부만 제거 (`logic.js` 무변경) |

### 9.1 롤백 판단 흐름

```
이팩트 배포 후 1게임 세션 KPI 확인
  └─ RAF 드롭 3회↑ → EFFECT_ENABLED = false (즉시 롤백)
  └─ slowMo 복귀 실패 → slowMo 기능만 비활성화
  └─ 정상 → 유지
```

## 10. Edge Case / 실패 케이스

| 케이스 | 상황 | 예상 동작 |
|--------|------|----------|
| **EC-1** | 8× 섭취 후 곧바로 게임 종료(벽 충돌) | 이팩트 즉시 중단, Game Over 오버레이 정상 표시 |
| **EC-2** | 이팩트 실행 중 Space 눌러 일시정지 | `status === "paused"` 감지 → RAF 중단 → 이팩트 애니메이션 동결 (resume 시 남은 시간만큼 재개) |
| **EC-3** | 이팩트 실행 중 Space 눌러 재시작 | `doRestart()` 호출 → `activeEffects` 초기화 + `TICK_MS` 원복 |
| **EC-4** | 연속으로 4× → 8× 빠르게 섭취 (중첩) | 두 이팩트 병렬 실행 허용, 슬로우 모션은 후발 8×가 타이머를 override |
| **EC-5** | 저성능 기기에서 파티클 24개 렌더링 | RAF 드롭 감지 시 파티클 수를 8개로 자동 감소 (`EFFECT_QUALITY = "low"` 폴백) |
| **EC-6** | `requestAnimationFrame` 미지원 환경 | 이팩트 비활성화, 기존 glow 렌더링만 유지 |
| **EC-7** | CPU 지렁이가 8× 아이템 섭취 | 이팩트 없음 (CPU 섭취는 이팩트 미발동, 명세 §4 트리거 조건) |

## 11. 사용자 시나리오

### 시나리오 A. 기본 플레이 중 1× 아이템 섭취

1. 플레이어가 황금색(1×) 아이템을 향해 이동한다.
2. 아이템 셀에 도달하면 `playerAteFood = true` 트리거 발동.
3. 섭취 셀 위치에서 황금빛 동심원 파문 2개가 400ms 동안 외부로 퍼지며 사라진다.
4. 게임은 중단 없이 계속 진행된다 (새 food spawn, 점수 +10, 길이 +1).

### 시나리오 B. 고배수 8× 아이템 섭취 전체 연출

1. 플레이어가 빨간색(8×) 아이템에 접근한다.
2. 섭취 순간:
   - 전체 화면에 빨간 오버레이 플래시(0 → 0.4 → 0, 300ms) 발생.
   - 섭취 위치에서 빨간 대형 파티클 24개가 방사형 폭발.
   - 게임 틱 속도가 240ms로 느려짐 (300ms간 슬로우 모션).
   - 캔버스가 ±3px 진동 (200ms, 2회).
3. 300ms 후 TICK_MS가 120ms로 복귀, 게임 정상 속도 재개.
4. 점수 +80, 길이 +8 적용.

### 시나리오 C. 이팩트 중 일시정지

1. 플레이어가 4× 아이템을 섭취 — 보라 플래시 + 파티클 발동 (700ms).
2. 이팩트 실행 중(t=300ms) 플레이어가 Space를 누른다.
3. RAF가 즉시 중단되고 이팩트 애니메이션이 t=300ms 프레임에서 동결된다.
4. PAUSED 오버레이가 표시된다.
5. 플레이어가 Space를 다시 눌러 재개 → 이팩트 남은 400ms 재개 + 게임 루프 재시작.
