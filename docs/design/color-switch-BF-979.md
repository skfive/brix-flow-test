## 1. 시안 개요

### 1.1 게임 개념 — 색·문자 간섭(Stroop) 반응 게임

화면 중앙에 **색 이름 단어**(예: "빨강")가 **특정 잉크 색**으로 표시된다. 단어의 **뜻**과 **잉크 색**이 일부러 다를 수 있다(간섭 = Stroop 효과). 매 라운드 상단의 **규칙(RULE)** 배너가 이번에 무엇을 골라야 하는지 지정한다:

| 규칙 | 표시(아이콘+텍스트) | 정답 기준 |
|---|---|---|
| **글자 색** | 🎨 `글자 색` | 단어가 **칠해진 잉크 색** |
| **글자 뜻** | 💬 `글자 뜻` | 단어가 **의미하는 색 이름** |

규칙은 라운드마다 **전환(switch)** 될 수 있다 — 이것이 "컬러 스위치"의 반응·인지 전환 부하다. 플레이어는 4개의 색상 응답 버튼(빨강·초록·노랑·파랑) 중 규칙에 맞는 색을 **빠르고 정확하게** 선택한다.

### 1.2 화면 구성(단일 SPA)
- **HUD** — 점수 / 연속 정답(streak) / 남은 시간(또는 라운드 타이머)
- **규칙 배너(RULE)** — 이번 라운드의 판단 기준(색/뜻) + 아이콘
- **자극 존(STIMULUS)** — 색 단어 + 잉크 색 + 고정 도형 마커
- **반응 피드백(FEEDBACK)** — 정답 ✓ / 오답 ✗ (아이콘+텍스트+위치, 색 단독 금지)
- **응답 버튼 4색(2×2)** — 각 색 = 색상 + 텍스트 라벨 + 고정 도형 + 숫자키
- **컨트롤** — 시작 / 다시하기
- **조작 안내(hint)** — 키보드·색맹 안내

### 1.3 사용자 경험 목표
1. **규칙 전환을 즉시 인지** — "지금은 색? 뜻?"을 상단 규칙 배너의 아이콘+텍스트+굵기로 0.3초 내 파악.
2. **간섭 자극의 명료함** — 자극 단어의 크기·중앙 배치로 뜻을 읽되, 잉크 색은 도형 마커로 이중 표현하여 색맹도 잉크를 식별.
3. **반응 피드백 즉각성** — 정답/오답을 아이콘(✓/✗)+텍스트+짧은 모션(정답=팝, 오답=흔들림)으로 0.2초 내 전달. 색 단독 신호 금지.
4. **진척·성취감** — 점수·연속 정답 카운터가 정답 시 증가 애니메이션으로 반응.
5. **키보드 완전 조작** — 마우스 없이 숫자키 1~4 / Tab+Enter 로 응답·컨트롤 전부 조작.

### 2.3 색맹·명도 안전성 근거
- **4색은 색상만으로 구분하지 않는다:** 각 색은 (a) 고정 **도형 마커**(●▲■◆), (b) **텍스트 라벨**("빨강"…), (c) **숫자키**(1~4), (d) 응답 버튼의 **고정 위치**(2×2 사분면)로 **4중 식별**된다. 적록/청황 색각 이상에서도 색 구분에 의존하지 않는다.
- **자극 잉크 색도 도형으로 이중화:** 자극 단어 옆(또는 위)에 **잉크 색에 대응하는 도형 마커**를 항상 함께 렌더한다 → "글자 색" 규칙 라운드에서도 색각 이상 사용자가 잉크 색을 도형으로 식별 가능. 단어의 **뜻**과 **잉크 도형**이 서로 달라 간섭 효과(게임성)는 그대로 유지된다.
- **피드백은 색 단독 금지:** 정답=`✓`+"정답"+팝 모션, 오답=`✗`+"오답"+흔들림 모션. 색은 보조 강화일 뿐 필수 채널이 아니다.
- **명도 대비:** 4색 lit 값은 모두 다크 배경(`#0B0F17`) 대비 큰 명도차를 가지며, 응답 버튼 라벨 텍스트는 어두운 `--sw-*-dim` 표면 위 `#E8EDF4`(≈ 12:1↑)로 WCAG AA 이상.

## 5. 컴포넌트 명세

### 5.1 HUD (`.hud`)
- 3개 `.hud__item`(점수 / 연속 / 시간). 각 항목 = `--text-hud-label`(라벨) + `--text-hud-value`(값, mono).
- **상태/인터랙션:** 점수·연속은 정답 시 값 갱신 + `pop` 애니메이션(scale 1→1.12→1, `--motion-fast`). 시간은 mono 카운트다운. 오답 시 연속(streak)은 0 으로 리셋되며 라벨 옆에 `↺` 표기(색 무관 신호).
- **props(dev 계약):** `score:number`, `streak:number`, `timeLeft:string(mm:ss 또는 s)`.

### 5.2 규칙 배너 (`.rule-banner`)
- 아이콘(🎨 색 / 💬 뜻) + `--text-rule` 라벨 + 강조 배경(`--rule-ink-bg`).
- **상태:** `data-rule="ink"`(글자 색) / `data-rule="word"`(글자 뜻). 규칙 전환 시 배너에 좌→우 슬라이드+강조 플래시(120ms)로 "바뀜"을 알림.
- **접근성:** `role="status"` `aria-live="polite"` — 규칙 전환을 스크린리더에 공지. 아이콘은 장식이 아니라 텍스트 라벨과 함께여서 색 무관.
- **props:** `rule:'ink'|'word'`.

### 5.3 자극 존 (`.stimulus`)
- 중앙 대형 색 단어(`--text-stimulus`) + **잉크 색 도형 마커**(단어 앞, `.stimulus__mark`). 예: 뜻="빨강", 잉크=파랑 → `◆ 빨강`(◆·파랑색, 글자 "빨강"도 파랑 잉크).
- **간섭 표현:** 단어 텍스트 색 = 잉크 색(`--sw-*-lit`). 도형 마커 색·모양 = 잉크 색 대응. 단어의 뜻(문자열)은 잉크와 독립 → Stroop 간섭 유지.
- **props:** `word:'빨강'|'초록'|'노랑'|'파랑'`, `ink:'red'|'green'|'yellow'|'blue'`.
- **상태:** 기본 / `is-correct`(배경 `--feedback-ok-tint` + 팝) / `is-wrong`(배경 `--feedback-ng-tint` + 흔들림).

### 5.4 반응 피드백 (`.feedback`)
- 자극 존 하단 라인. **정답:** `✓` 아이콘 + "정답" + 획득 점수(`+80`). **오답:** `✗` 아이콘 + "오답" + 정답 안내("정답: ▲ 초록").
- **색 단독 금지:** 아이콘 모양(✓/✗)과 텍스트가 1차 채널, `--color-success/--color-danger` 는 보조. 무채색 환경에서도 ✓/✗ 로 구분.
- **접근성:** `role="status"` `aria-live="assertive"` — 결과 즉시 공지.
- **모션:** 정답 팝(scale), 오답 흔들림(translateX ±6px, 2회). `prefers-reduced-motion` 시 모션 제거하고 아이콘·텍스트·틴트만 표시.
- **props:** `result:'correct'|'wrong'|null`, `gained:number`, `answerHint:string`.

### 5.5 응답 버튼 (`.answer-btn`) × 4
- 구조: `[도형 마커] [색 이름] [숫자키 배지]`. 배경 = `--sw-*-dim`, 도형·강조 = `--sw-*-lit`.
- **상태:**
  - 기본(dim), `:hover`(brightness 1.1 — 데스크톱만), `:focus-visible`(2px `--color-focus-ring` outline + offset),
  - `:active`/선택 순간(scale .96 + lit 글로우),
  - `is-answered-correct`(선택이 정답 → lit + ✓ 오버레이), `is-answered-wrong`(선택이 오답 → 흔들림 + ✗ 오버레이 + 정답 버튼에 ✓ 하이라이트),
  - `[aria-disabled="true"]`(라운드 종료·게임오버 시 입력 잠금, cursor:not-allowed).
- **접근성:** `<button type="button" aria-label="빨강 (도형 원, 숫자 1)">`. 숫자키 1~4 로도 트리거.
- **props:** `color:'red'|'green'|'yellow'|'blue'`, `state:'idle'|'answered-correct'|'answered-wrong'|'disabled'`.

### 5.6 컨트롤 (`.controls`)
- `시작`(primary, accent) / `다시하기`(secondary). 게임 진행 중 `시작` 비활성, 대기·게임오버 시 `다시하기` 활성.
- **상태:** 대기(시작 활성/다시하기 비활성) → 진행(둘 다 상황따라) → 게임오버(다시하기 활성 강조).

### 5.7 게임오버 배너 (`.gameover`)
- 시간 종료 시 자극 존을 덮는 결과 요약: 최종 점수·최고 연속·정답률 + `다시하기` CTA. `role="status" aria-live="polite"`.

### 5.8 색맹 안내 배지 (`.a11y-badge`)
- 앱 타이틀 우측 `ⓘ 색·도형·숫자` pill — 4색이 색 외 채널로도 구분됨을 상시 고지.
