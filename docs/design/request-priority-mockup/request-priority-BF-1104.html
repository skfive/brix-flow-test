요청 우선순위 분류

영향도(Impact) × 긴급도(Urgency) 를 선택하면 P1~P4 우선순위와 다음 조치가 즉시 계산됩니다. (로컬 상태 전용 데모)

상태 시뮬레이션 A — resolved (P1)

입력

영향도 (Impact)
긴급도 (Urgency)

결과

P1 긴급(Critical)

Critical × Immediate

다음 조치

즉시 담당자 배정 + 실시간 모니터링 채널 개설, 필요 시 경영진/책임자 에스컬레이션

목표 착수 시간: 15분 이내

상태 시뮬레이션 B — empty (미선택)

입력

영향도 (Impact)
긴급도 (Urgency)

결과

P?

두 축을 모두 선택하세요. (미선택 시 우선순위를 계산하지 않습니다.)

상태 시뮬레이션 C — error (정의되지 않은 등급)

결과

정의되지 않은 등급

매트릭스에 없는 등급 코드입니다. 계산을 수행하지 않습니다. (4×4 외 값으로 폴백 추정하지 않음)

P1~P4 상태 배지 + 다음 조치 영역 비교

P1 — 긴급(Critical)

P1 긴급(Critical)

다음 조치

즉시 담당자 배정 + 실시간 모니터링 채널 개설, 필요 시 경영진/책임자 에스컬레이션

목표 착수 시간: 15분 이내

P2 — 높음(High)

P2 높음(High)

다음 조치

당일 내 담당자 배정 및 착수, 2시간 주기 진행 상황 업데이트

목표 착수 시간: 4시간 이내

P3 — 보통(Medium)

P3 보통(Medium)

다음 조치

익영업일 이내 착수, 정기(일 단위) 업데이트

목표 착수 시간: 1영업일 이내

P4 — 낮음(Low)

P4 낮음(Low)

다음 조치

백로그에 등록, 다음 스프린트 계획 시 우선순위 재검토

목표 착수 시간: 스프린트 계획 반영

참조 — 우선순위 매트릭스 (현재 선택: Critical × Immediate 하이라이트)

Impact × Urgency → P1~P4

Impact \ UrgencyImmediateHighMediumLow
Critical P1 P1 P2 P2
High P1 P2 P2 P3
Medium P2 P3 P3 P4
Low P3 P4 P4 P4
