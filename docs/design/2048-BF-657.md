## 1. 시안 개요

### 1.1 변경 범위

- 신규 SPA 모듈 `game-2048/index.html` (단일 페이지 게임)
- 4×4 그리드 보드 — 슬라이딩 타일 + 합치기 인터랙션
- 타일 값 기반 11단계 컬러 시스템 (2 → 2048)
- 점수판 (현재 점수 + 베스트 점수)
- 타일 합치기 모션 (slide + pop 애니메이션)
- 게임 오버 / WIN 오버레이 모달
- 반응형 레이아웃 (desktop / mobile 2단계)

### 1.2 사용자 경험 목표

| 목표 | 세부 내용 |
|---|---|
| **기존 게임 모듈과 명확한 차별화** | 전체 다크 배경 일색인 다른 모듈과 달리 **라이트-퍼스트 Warm Sand 팔레트** 채택 — 페이지를 열자마자 다른 감성임을 즉시 인지 |
| **타일의 숫자 성장이 시각적으로 체감됨** | 2(크림/베이지) → 64(붉은 코랄) → 2048(황금 글로우) 11단계 컬러 진화 — 수를 합치는 쾌감이 색 변화로 증폭 |
| **컴팩트한 퍼즐 카드 감성** | 전체 페이지를 채우는 다른 게임과 달리, 보드는 **중앙 플로팅 카드** 형태 — 테트리스/스네이크의 아케이드 레이아웃과 확연히 다름 |
| **타일 이동의 물리감** | slide + merge pop 애니메이션으로 타일이 실제로 밀리고 합쳐지는 물리감 표현 |
| **점수 피드백** | 합치기 발생 시 `+득점` 팝업이 타일 위에서 위로 올라가며 사라짐 |
| **접근성** | 방향키 + WASD 조작 + Tab/Enter 오버레이 조작 |

### 1.3 기존 모듈과의 차별화 대비표

| 디자인 요소 | 기존 게임 모듈 (snake/tetris/rps/baseball) | 2048 (이 명세) |
|---|---|---|
| **배경 테마** | 다크 우선 (`#0F1115` ~ `#0A0C10`) | **라이트 우선** (`#FAF8EF`) |
| **주요 색조** | 쿨 네이비 + 네온 (시안/초록/파랑) | **워밍 샌드 + 앰버/코랄** |
| **레이아웃** | 전체 페이지 / 3단 HUD 분리 | **컴팩트 중앙 카드 (최대 480px)** |
| **점수 위치** | 별도 HUD 패널/사이드바 | 보드 위 인라인 헤더 |
| **타입 느낌** | 아케이드/SF (상단 게임 타이틀, 네온 HUD) | **퍼즐/보드게임 (차분한 타이포)** |
| **타일 컬러** | 단색 네온 피스 | **값에 따라 진화하는 11단계 그라데이션 색** |

### 1.4 비목표 (Out of Scope)

- 터치 스와이프 컨트롤 (별도 Epic)
- 멀티플레이어·온라인 랭킹
- 효과음 (별도 Epic)
- 4096+ 타일 특수 처리 (2048 초과 타일은 동일 스타일 유지)
- 애니메이션 비활성화 설정 (v1은 고정)

--font-sans: ui-sans-serif, system-ui, -apple-system, "Segoe UI", sans-serif;
--font-mono: ui-monospace, Menlo, Consolas, monospace;

## 4. 레이아웃

### 4.1 전체 구조 개요

기존 모듈의 전체 페이지 레이아웃과 다르게,  
2048은 **컴팩트 중앙 카드** — 페이지 중앙에 하나의 카드가 뜨는 형태.

```
┌──────────────────────────────────────────────────┐
│  전체 페이지 배경 (--color-bg-canvas)             │
│                                                  │
│     ┌────────────────────────────────────┐       │
│     │           게임 카드 (max 480px)     │       │
│     │  ┌──────────────────────────────┐  │       │
│     │  │  2048 [타이틀]   [🌙 테마]   │  │       │
│     │  ├──────────────────────────────┤  │       │
│     │  │  [SCORE: 0]  [BEST: 0] [NEW] │  │       │
│     │  ├──────────────────────────────┤  │       │
│     │  │ "타일을 합쳐서 2048을 만드세요"│  │       │
│     │  ├──────────────────────────────┤  │       │
│     │  │                              │  │       │
│     │  │   ┌──── 4×4 보드 ────┐       │  │       │
│     │  │   │ [  ] [  ] [2 ] [4] │    │  │       │
│     │  │   │ [  ] [8 ] [16] [  ]│    │  │       │
│     │  │   │ [4 ] [32] [  ] [8] │    │  │       │
│     │  │   │ [  ] [2 ] [4] [16] │    │  │       │
│     │  │   └────────────────────┘     │  │       │
│     │  │                              │  │       │
│     │  └──────────────────────────────┘  │       │
│     └────────────────────────────────────┘       │
│                                                  │
└──────────────────────────────────────────────────┘
```

> **차별화 포인트**: snake/tetris는 전체 화면을 꽉 채우는 아케이드 레이아웃.  
> 2048은 캔버스 중앙에 카드 1개 — 보드게임 감성.

### 4.2 게임 카드 치수

| 브레이크포인트 | 카드 최대 폭 | 패딩 | 보드 크기 |
|---|---|---|---|
| Desktop ≥ 520px | `480px` | `24px` | `4 × 타일크기 + 5 × gap` |
| Mobile < 520px | `100vw - 24px` | `12px` | `4 × 타일크기 + 5 × gap` |

카드 스타일:
- `background: var(--color-bg-surface)`
- `border-radius: 16px`
- `box-shadow: 0 4px 24px rgba(0,0,0,0.10)` (light) / `0 4px 24px rgba(0,0,0,0.40)` (dark)
- `padding: var(--space-5)` (24px)

### 4.3 4×4 그리드 보드 치수

| 브레이크포인트 | 타일 크기 | 보드 크기 (타일 4 + 갭 5) |
|---|---|---|
| Desktop ≥ 520px | `100px × 100px` | `460px` (4×100 + 5×12) |
| Mobile < 520px | `72px × 72px` | `336px` (4×72 + 5×12) |

보드 컨테이너:
- `background: var(--board-bg)`
- `border-radius: var(--board-radius)` (8px)
- `padding: var(--board-gap)` (12px) — 외곽도 갭 동일
- `display: grid; grid-template-columns: repeat(4, 1fr); gap: var(--board-gap)`

### 4.4 카드 헤더 영역

```
┌─────────────────────────────────────────────┐
│  2048              [SCORE]  [BEST]  [🌙]    │
│  타이틀(48px/800)   숫자   숫자   토글         │
└─────────────────────────────────────────────┘
```

- 헤더: `display: flex; align-items: flex-end; justify-content: space-between`
- 타이틀 "2048": color `var(--tile-64)` (`#F65E3B`) — 타일 컬러와 연결
- 점수 그룹: `display: flex; gap: var(--space-3)` (12px)
- 각 점수 박스: `background: var(--board-bg); border-radius: 6px; padding: 8px 12px; min-width: 80px; text-align: center`

### 4.5 서브 안내 텍스트

- "타일을 합쳐서 **2048**을 만드세요!" (bold 는 2048만 강조)
- `font-size: 13px; color: var(--color-text-secondary); margin: var(--space-2) 0 var(--space-3)`
- **[새 게임]** 버튼은 안내 텍스트 우측에 인라인 배치

### 4.6 반응형 상세

#### Desktop (≥ 520px)

- 게임 카드 `max-width: 480px`, 페이지 중앙 (`margin: auto; padding: 40px 12px`)
- 타일 크기 100×100, 타일 숫자 40–26px
- 헤더 타이틀 48px

#### Mobile (< 520px)

- 카드 전폭 (좌우 패딩 12px)
- 타일 크기 72×72, 타일 숫자 28–18px
- 헤더 타이틀 36px
- 점수 박스 min-width 64px

## 5. 컴포넌트 명세

### 5.1 타일 (Tile)

**구조**:
```html
<div class="tile" data-value="2" style="--tile-x: 0; --tile-y: 0;">
  <span class="tile__num">2</span>
</div>
```

CSS 포지셔닝은 CSS 변수(`--tile-x`, `--tile-y`)로 `translate` 제어:

```css
.tile {
  position: absolute;
  width: var(--tile-size);
  height: var(--tile-size);
  border-radius: var(--tile-radius);
  display: flex;
  align-items: center;
  justify-content: center;
  transform: translate(
    calc(var(--tile-x) * (var(--tile-size) + var(--board-gap))),
    calc(var(--tile-y) * (var(--tile-size) + var(--board-gap)))
  );
  transition: transform var(--slide-duration) var(--slide-easing);
}
```

**값별 스타일 (`data-value` attribute selector)**:

| `data-value` | background | color |
|---|---|---|
| `"2"` | `var(--tile-2)` = `#EEE4DA` | `#776E65` |
| `"4"` | `var(--tile-4)` = `#EDE0C8` | `#776E65` |
| `"8"` | `var(--tile-8)` = `#F2B179` | `#F9F6F2` |
| `"16"` | `var(--tile-16)` = `#F59563` | `#F9F6F2` |
| `"32"` | `var(--tile-32)` = `#F67C5F` | `#F9F6F2` |

## 6. 타일 합치기 모션 명세

### 6.1 슬라이드 모션 (Slide)

방향키 입력 시 타일이 이동하는 모션:

| 토큰명 | 값 | 용도 |
|---|---|---|
| `--slide-duration` | `100ms` | 타일 이동 시간 |
| `--slide-easing` | `ease-in-out` | 이동 곡선 |

구현 방식: CSS `transform: translate()` + `transition` 조합
- JS에서 새 위치 계산 → CSS 변수 (`--tile-x`, `--tile-y`) 업데이트 → CSS transition 자동 처리
- 이동 완료 후 (`transitionend`) DOM 정리 (합쳐진 타일 제거)

### 6.2 합치기 팝 모션 (Merge Pop)

두 타일이 합쳐질 때 새 타일에 적용:

| 토큰명 | 값 |
|---|---|
| `--merge-duration` | `200ms` |
| `--merge-easing` | `ease-out` |

```css
@keyframes tileMergePop {
  0%   { transform: translate(var(--tx), var(--ty)) scale(1.0); }
  40%  { transform: translate(var(--tx), var(--ty)) scale(1.18); }
  100% { transform: translate(var(--tx), var(--ty)) scale(1.0); }
}
.tile--merged {
  animation: tileMergePop var(--merge-duration) var(--merge-easing);
}
```

### 6.3 신규 타일 등장 모션 (Spawn)

이동 완료 후 새로 생성되는 타일 (2 또는 4):

```css
@keyframes tileSpawn {
  0%   { transform: translate(var(--tx), var(--ty)) scale(0); }
  60%  { transform: translate(var(--tx), var(--ty)) scale(1.1); }
  100% { transform: translate(var(--tx), var(--ty)) scale(1.0); }
}
.tile--new {
  animation: tileSpawn 200ms ease-out;
}
```

딜레이: 슬라이드(`100ms`) + 합치기(`200ms`) 이후 → `animation-delay: 120ms` 적용 권장

### 6.4 모션 타임라인 요약

```
[방향키 입력]
  │
  ├── 0ms: 타일 위치 계산 (JS)
  ├── 0ms: CSS transition 시작 (slide, 100ms)
  ├── 100ms: slide 완료 / 합치기 타일 생성
  ├── 100ms: merge pop 시작 (200ms)
  ├── 120ms: 신규 타일 spawn 시작 (딜레이 120ms, 200ms)
  └── 320ms: 전체 애니메이션 완료 → 다음 입력 가능
```

| **컴포넌트 매핑** — 모든 UI 컴포넌트(타일/점수/오버레이)가 명세됨 | ✅ | §5.1~5.7 완전 명세; 슈퍼 타일(4096+) 처리 포함 |

### 잠재적 주의사항

1. **타일 CSS 포지셔닝 복잡도**: `position: absolute` + CSS 변수 `translate` 방식은 구현 난이도가 있음. dev는 §10.5 JS 패턴 참조 필수.
2. **애니메이션 시퀀스**: slide(100ms) → merge/spawn이 정확히 맞지 않으면 타일이 겹쳐 보임. §6.4 타임라인 엄수 권고.
3. **다크 테마에서 타일 컬러**: 타일 11단계 컬러는 라이트/다크 무관하게 동일 HEX 사용. 다크 배경에서도 타일 컬러는 충분히 대비됨 (확인).
4. **localStorage `game2048:board`**: 자동 저장 기능은 UX 향상이지만 세션 복구 로직 필요. dev 판단에 위임.
