레이아웃

최대 폭 1080px 중앙 정렬. 카드 그리드는 repeat(auto-fit, minmax(320px, 1fr)). breakpoint 640px 이하에서 파이프라인은 세로 스택, 화살표 숨김. spacing 은 4px 배수 토큰(--space-1~--space-6) 사용.

컴포넌트 명세

  • .badge — variant: pass | pending | fail | na. 좌측 점 인디케이터 + 라운드 pill. 상태 컬러 토큰 매핑.
  • .stage — state: current | done | (기본). current 는 primary 테두리+배경 강조.
  • .card — head(제목 + .cat-label) + ul.checklist(항목별 상태 배지).
  • .btn — variant: primary | ghost. hover: brightness/배경, focus-visible: accent outline 2px.

인터랙션 상태 예시

hover / focus-visible 상태는 실제 CSS 로 표현됨

dev 구현 가이드

  1. :root 에 위 컬러/스페이싱 토큰 CSS 변수 정의 (하드코딩 색상 금지 — 토큰 참조).
  2. 상태 배지는 단일 .badge 클래스 + variant 모디파이어로 구현.
  3. 파이프라인 현재 단계는 .stage.current 로만 강조 — 로직은 데이터 바인딩으로 분리 권장.
  4. 접근성: 섹션 aria-labelledby 유지, 상태를 색상만으로 표현하지 말고 텍스트 라벨 병기.
  5. 반응형 640px breakpoint 준수.
